## 競合比較と差別化

### AirXpress（人手による運営代行）との比較

| 機能 | AirXpress（人手） | Minpaku DX（AI） |
|---|---|---|
| ゲストメッセージ対応 | 多言語スタッフが手動対応 | AI自動返信 + オーナー承認 |
| 対応速度 | 営業時間内（深夜は遅延） | 24時間即時対応 |
| 多言語対応 | 人件費で対応（英/中/韓） | AIが自動検出・自動翻訳（無制限言語） |
| 清掃手配 | 自社スタッフ/提携先 | チェックアウト検知→自動手配 |
| 価格最適化 | ダイナミックプライシング（人が設定） | AI自動価格調整 |
| リマインダー | 手動送信 | 予約タイムライン連動の自動送信 |
| 宿泊者情報回収 | スタッフが回収 | 自動フォーム送信・回収 |
| 料金体系 | 売上の5〜20% | 月額固定（¥3,980〜） |
| スケーラビリティ | スタッフ増員が必要 | AI処理のため物件数に制限なし |

### Minpaku DXの差別化ポイント

1. **コスト構造が根本的に違う** — 人件費ゼロ。物件数が増えてもコストは微増
2. **24時間無休** — 深夜3時のゲスト問い合わせにも即座にAI返信
3. **物件知識のRAG** — 物件ごとのルール・設備・FAQをAIが完全に把握
4. **オーナーの主体性を維持** — 代行に丸投げではなく、AIドラフト→承認のフロー

## 展開媒体・アーキテクチャ

### 媒体選定：LINE + Web のハイブリッド

| 役割 | 媒体 | 理由 |
|---|---|---|
| **日常オペレーション** | LINE Bot / LIFF | 通知速度最速、導入ハードルゼロ、外出先対応 |
| **管理・設定** | Web（PWA対応） | リッチUI、物件ルール編集、分析ダッシュボード |

**ネイティブアプリは作らない。** PMF確認前にコスト大、LINEで十分なUXを実現できる。

#### 1-2. DBスキーマ

```sql
CREATE TABLE owners (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  line_user_id TEXT,
  beds24_refresh_token TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE properties (
  id SERIAL PRIMARY KEY,
  owner_id UUID REFERENCES owners(id),
  beds24_property_id INT NOT NULL,
  name TEXT NOT NULL,
  rules_content TEXT,
  reply_tone TEXT DEFAULT 'polite',
  signature TEXT DEFAULT '民泊スタッフ一同',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE message_logs (
  id SERIAL PRIMARY KEY,
  property_id INT REFERENCES properties(id),
  booking_id INT NOT NULL,
  guest_message TEXT NOT NULL,
  detected_language TEXT,
  category TEXT,
  urgency TEXT CHECK (urgency IN ('high', 'medium', 'low')),
  ai_draft TEXT NOT NULL,
  final_reply TEXT,
  action TEXT CHECK (action IN ('sent', 'edited', 'skipped', 'auto_sent')),
  edit_distance INT,
  responded_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE scheduled_messages (
  id SERIAL PRIMARY KEY,
  property_id INT REFERENCES properties(id),
  booking_id INT NOT NULL,
  trigger_type TEXT NOT NULL,  -- 'pre_checkin', 'post_checkin', 'pre_checkout', 'post_checkout'
  trigger_offset_hours INT,    -- トリガー時点からの時間差
  message_template TEXT NOT NULL,
  sent_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE cleaning_tasks (
  id SERIAL PRIMARY KEY,
  property_id INT REFERENCES properties(id),
  booking_id INT NOT NULL,
  checkout_date DATE NOT NULL,
  status TEXT CHECK (status IN ('pending', 'notified', 'confirmed', 'completed')),
  cleaner_contact TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 技術スタック

| レイヤー | 技術 | 理由 |
|---|---|---|
| バックエンドAPI | FastAPI (Python) | 既存コード資産を活かせる、async対応 |
| DB / Auth | Supabase (PostgreSQL) | 無料枠あり、Auth/Realtime内蔵 |
| フロントエンド | Next.js + Tailwind CSS | SSR、Vercelで簡単デプロイ |
| AI | Google Gemini 2.5 Flash | コスパ最高、日本語性能が高い |
| メッセージング | LINE Messaging API + LIFF | ターゲット層に最適 |
| ホスティング | Railway (API) + Vercel (Web) | 低コスト、スケーラブル |
| ジョブキュー | APScheduler or Celery + Redis | 定期ポーリング・スケジュール送信 |
| 決済 | Stripe | サブスクリプション管理 |

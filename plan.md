# Minpaku DX — 開発ロードマップ

**最終更新**: 2026-03-07
**目標**: Beds24上の民泊メッセージをAIが下書きし、オーナーが1タップで承認・送信できる「Human-in-the-loop」運営システムの完成

---

## 現状の整理

### 完成済み
- [x] Beds24 API認証（invite code → refresh token → access token）
- [x] Beds24からメッセージ一覧取得（`source: guest` / `read: false` の絞り込み）
- [x] Gemini APIによるAI返信案の生成（`gemini-2.0-flash-lite`）

### 現状の問題
- APIキー・トークンがソースコードにハードコードされている（セキュリティリスク）
- AIが物件固有の知識を持っていない（WiFi、鍵の場所、ゴミ出しルール等が未反映）
- 返信を**実際にBeds24へ送信する機能がない**（生成して表示するだけ）
- 会話の前後の文脈をAIに渡せていない（直近1メッセージだけ）
- `import requests.py` という不正なファイルが存在する
- `main.py`と`ai_reply.py`が機能重複している

## フェーズ2: RAG — 物件別ナレッジベースの構築

**目標**: AIが物件ごとのルールを知った上で返信できるようにする

背景: 現状のAIは「一般的な民泊の知識」しかない。物件固有の情報（騒音モニターの説明、WiFiパスワード、ゴミ出し曜日、住所など）をRAGとして注入することで、実際に使える返信を生成できる。

- [ ] `rules/` ディレクトリを作成
- [ ] 物件ごとのMarkdownファイルを作成（例: `rules/property_206100.md`）
  ```
  # 平井戸建 (propertyId: 206100)
  ## 基本情報
  - 住所: 東京都江戸川区平井1-18-7
  - チェックイン: 16:00〜23:00 / チェックアウト: 10:00
  ## 設備
  - WiFi: SSID: Hirai_Guest / PASS: （オーナーに確認）
  - 騒音モニター: テレビ横に設置。会話は録音せず騒音レベルのみ計測。90db超でアラーム。
  ## ルール
  - 禁煙（玄関前も含む）
  - ゴミは室内にまとめ、外の集積所には出さない（清掃員が処理）
  ## Amazon配送先
  - 〒132-0035 東京都江戸川区平井1-18-7 102号室
  ```
- [ ] `ai_reply.py` を更新: `propertyId` でルールファイルを読み込み、システムプロンプトに注入する
- [ ] `rules/` ディレクトリに既存全物件のMarkdownを揃える

**完了条件**: 騒音モニターに関する質問に対して、AIが物件固有の説明を含む返信を生成できる

## フェーズ5: Human-in-the-Loop CLI（承認フロー）

**目標**: AIの返信案を人間が確認・編集してから送信する対話型CLIを構築

これがこのシステムの**最重要機能**。AIが誤った情報を送らないための安全装置であり、差別化ポイント。

- [ ] インタラクティブCLIフローを実装:
  ```
  ========================================
  【1件目】予約ID: 73705556 | Jeanne Yao
  チェックイン: 2025-12-26

  ▼ ゲストのメッセージ
  What is this alarm for?

  ▼ AI返信案
  ご不便をおかけして申し訳ありません。テレビ横の
  装置は騒音モニターで、会話は録音しておりません。
  ご安心ください。 — 民泊スタッフ一同

  [s]送信 / [e]編集 / [n]スキップ > _
  ```
- [ ] `s` → そのまま送信
- [ ] `e` → 編集モード（テキストを書き直せる）
- [ ] `n` → スキップ（未読のまま）
- [ ] 送信後、Beds24上でメッセージを既読にマークする

**完了条件**: CLIのみで1件のゲストメッセージへの返信が完結する
